##### 创建表

用于创建数据库中的表

```mysql

(
列名称1 数据类型
列名称2 数据类型

);
```

##### 排序

  order by语句用于根据指定的列对结果集进行排序，默认按照**升序**对记录进行排序；用desc关键字是按照**降序**进行排序**（by后面多个参数用逗号连接，不能用and）**


默认排序为asc升序，desc为降序

MySQL的`ORDER BY`实现特点：

1. **没有ROWID概念**（不像Oracle、PostgreSQL）
2. **使用filesort算法**：如果没有合适的索引，MySQL会使用临时文件排序
3. **快速排序算法**：不稳定排序算法，相同键值的相对顺序可能变化
4. **内存排序与磁盘排序**：根据数据大小选择不同算法

##### 清除表数据

如果我们仅仅需要除去表内的数据，但不删除表本身，就可以使用turncate table命令（仅仅删除表格中的数据）：

```mysql
turncate table 表名;
```

* 实例

    * 删除名为persons的表

    ```mysql
    turncate table persons;
    ```



补：turncate和delete的区别

| 特性           | delete                   | turncate                   |
| -------------- | ------------------------ | -------------------------- |
| 命令类型       | DML（数据操作语言）      | DDL（数据定义语言）        |
| 删除方式       | 逐行删除，记录每行日志   | 一次性删除所有数据页       |
| 性能           | 较慢（逐行操作）         | 极快（直接释放数据页）     |
| 事务日志       | 记录详细日志，可回滚     | 记录最少日志，通常不可回滚 |
| 触发器         | 会触发DELETE触发器       | 不会触发触发器             |
| 自增列重置     | 不重置自增计数器         | 重置自增计数器为初始值     |
| 锁粒度         | 行级锁（可能升级为表锁） | 表级锁（短暂独占）         |
| 外键约束       | 受外键约束影响           | 通常需要禁用外键约束       |
| 恢复数据       | 可通过事务日志恢复       | 通常需要禁用外键约束       |
| 是否可条件删除 | 支持where子句            | 只能删除全部数据           |
| 磁盘空间       | 不立即释放，标记为可重用 | 立即释放给操作系统         |

补：

* 回滚操作：rollback;

* 事务日志：数据库记录所有数据更改操作的文件，用于保证数据库的ACID特性（原子性，一致性，隔离性，持久性）；主要作用有：

    * 恢复机制：数据库崩溃之后，通过重放日志恢复到一致状态

    * 回滚操作：支持事务回滚到之前的状态

    * 数据复制：主从复制、Always on等基于日志

    * 审计追踪：记录谁在什么时候做了什么

* 查看日志的方法：

```mysql
##查看当前二进制日志状态
show master status;

##查看所有二进制日志文件
show binary logs;
```

* 触发器：触发器是一种特殊的存储过程，在特定事件（insert，update，delete）发生时自动执行
    * 类型：
        * **BEFORE/AFTER触发器**：在操作之前或之后触发
        * **INSTEAD OF触发器**：代替原操作执行
        * **ROW-LEVEL触发器**：每行触发一次
        * **STATEMENT-LEVEL触发器**：每条语句触发一次
    * 作用：
        * **数据审计**：记录所有数据变更
        * **数据校验**：复杂的业务规则验证
        * **数据同步**：自动更新相关表
        * **计算字段**：自动维护统计字段
        * **访问控制**：限制某些操作

* 锁粒度：指锁定的数据范围大小，影响并发性能和资源占用

```mysql
-- 行级锁
-- mysql默认使用行级锁，只锁定要操作的行，其他行可访访问

UPDATE users SET status = 'active' WHERE id = 1;
-- 只锁定id=1的行，id=2的行可以同时被其他事务修改

-- 页级锁
-- 锁定整个数据页（通常是8KB），包含多行，如果页中有100行，锁定时其他事务不能访问这100行

-- 表级锁
LOCK TABLES users WRITE;  -- 锁定整个表
UPDATE users SET status = 'active' WHERE id = 1;
-- 整个users表被锁定，其他事务不能访问
UNLOCK TABLES;


-- 查看锁信息
show engine innodb status\G;
```

| 锁粒度     | 并发性 | 开销 | 死锁概率 | 适用场景           |
| :--------- | :----- | :--- | :------- | :----------------- |
| **行级锁** | 高     | 大   | 高       | OLTP系统，高并发   |
| **页级锁** | 中     | 中   | 中       | 中等并发           |
| **表级锁** | 低     | 小   | 低       | 报表查询，批量操作 |



* 外键约束：用于维护表间的引用完整性，确保子表的值必须在父表中存在

```mysql
-- 父表（部门）
CREATE TABLE departments (
    dept_id INT PRIMARY KEY AUTO_INCREMENT,
    dept_name VARCHAR(50) NOT NULL
);

-- 子表（员工）
CREATE TABLE employees (
    emp_id INT PRIMARY KEY AUTO_INCREMENT,
    emp_name VARCHAR(50) NOT NULL,
    dept_id INT,
    -- 创建外键约束
    CONSTRAINT fk_dept
    FOREIGN KEY (dept_id) 
    REFERENCES departments(dept_id)
    ON DELETE CASCADE    -- 删除部门时，自动删除员工
    ON UPDATE CASCADE    -- 更新部门ID时，自动更新员工部门ID
);
```

### 外键动作选项：

| 动作                      | 描述                                     |
| :------------------------ | :--------------------------------------- |
| **ON DELETE CASCADE**     | 删除父表记录时，同时删除子表相关记录     |
| **ON DELETE SET NULL**    | 删除父表记录时，将子表外键设为NULL       |
| **ON DELETE SET DEFAULT** | 删除父表记录时，将子表外键设为默认值     |
| **ON DELETE RESTRICT**    | 如果子表有引用，禁止删除父表记录（默认） |
| **ON UPDATE CASCADE**     | 更新父表主键时，同时更新子表外键         |
| **ON UPDATE SET NULL**    | 更新父表主键时，将子表外键设为NULL       |
