#### AS别名

通过使用SQL，可以为列名称和表名称指定别名（Alias），别名使查询程序更易阅读和书写

```mysql
-- 表别名
select 列名/* from 表名 as 别名;

-- 列别名
select 列名 as 别名 from 表名;
```

* 实例

    * 使用表名称别名（这么做是为了将表名指代为p，当遇到特别长的表名时，就可以用别名来替代原本的表名）

    ![image-20260130203600625](C:\Users\666\AppData\Roaming\Typora\typora-user-images\image-20260130203600625.png)

    **补**：“在单表查询时，因为只涉及一张表，可以直接使用列名，数据库能明确知道列来自哪里，所以通常不需要指定表名。

    但在多表查询时，情况就变了：

1. **消除歧义（必须）**：当多个表存在同名的列（比如 `user` 表和 `order` 表可能都有 `id` 列），如果不指明，数据库就无法判断你到底要哪个`id`。这时**必须**通过 `表名.列名` 来明确指定。

2. **简化书写（推荐）**：表名可能很长（例如 `customer_order_details`），在多表连接时反复书写会很繁琐。通过给表起一个简短的**别名**（Alias），可以极大简化SQL语句，使其更清晰易读。

3. **自连接等特殊场景（必须）**：在**自连接**中，我们需要将同一张表当作逻辑上的多张表来使用。如果不给表起不同的别名，数据库就无法区分你指的是“第一个实例”还是“第二个实例”中的列。因此，别名在这里是**必需的**。

    

    * 使用列名别名

    ![image-20260130204842918](C:\Users\666\AppData\Roaming\Typora\typora-user-images\image-20260130204842918.png)

    **另**：别名不仅可以用在表上，也可以用在列上（`SELECT name AS customer_name`），让查询结果的列标题更友好。给表起别名时，通常使用 `AS` 关键字（例如 `FROM users AS u`），但很多数据库也支持直接省略 `AS`（例如 `FROM users u`），这是一种更常见的简洁写法。”

#### JOIN多表关联

JOIN用于根据两个或多个表中的列之间的关系，从这些表中查询数据

有时我们为了得到完整的结果，需要从两个或更多的表中获取结果，我们就需要执行join

数据库中的表可通过键将彼此联系起来，主键（Primary Key）是一个列，在这个列中的每一行的值都是唯一的，在表中，每个主键的值都是唯一的，这样做的目的是在不重复每个表中的所有数据的情况下，把表间的数据交叉捆绑在一起

```mysql
select 列名 from 表A inner|left|right|full join 表B on 表A主键列 = 表B外键列; 
```

不同的SQL JOIN

1. join：如果表中有至少一个匹配，则返回行
2. inner join：内部连接，返回两表中匹配的行
3. left join：即使右表中没有匹配，也从左表返回所有的行
4. right join：即使左表中没有匹配，也从右表返回所有的行
5. full join：只要其中一个表中存在匹配，就返回行

#### 合并结果集

union操作符用于合并两个或多个select语句的结果集（就相当于是并集）

```mysql
select 列名 from 表A union (all) select 列名 from 表B;
```

union操作符默认为选取不同的值（去重），如果查询结果需要显示重复的值，就要加一个all（不去重就要加一个all）

**union结果集 中的列名总是等于union中第一个select语句中的列名**

#### VIEW视图

在SQL中，视图是基于SQL语句的结果集的可视化的表

视图包含行和列，就像一个真实的表，视图中的字段就是来自一个或多个数据库中的真实的表中的字段。我们可以向视图中添加SQL函数、where以及join的子句，我们也可以提交数据，就像这些来自于某个单一的表

```mysql
create view 视图名 as 
select 列名
from 表名
where 查询条件;
```

视图总是显示最近的数据，每当用户查询视图的时候，数据库引擎通过使用sql语句来重建数据

如果想要更新视图中的列或者其他信息，无需删除，使用create or replace view

```mysql
create or replace view 视图名 as
select 列名
from 表名
where 查询条件;
```

删除视图

```mysql
drop view persons_beijing;
```
